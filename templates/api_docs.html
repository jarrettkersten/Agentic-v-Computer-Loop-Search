Cora Code Investigator API

External API reference for integrating with the Cora Code Investigator. Use these endpoints to query the Cora PPM codebase programmatically from tools like Jira, n8n, or custom integrations.

Bearer Token — API key from Admin Dashboard or EXTERNAL_API_KEY env var Admin Login — Required for dashboard endpoints Public — Health & branch listing
Query
POST /api/query Submit a query (async) Bearer

The primary endpoint for external integrations (Jira, n8n, custom tools). Submits a natural language query and returns a job_id immediately. Poll /api/query/{job_id} for results. Optionally include a screenshot — Claude will extract UI context and append it to the query, just like the web UI. Requires a Bearer token matching the EXTERNAL_API_KEY environment variable.

Headers

NameValueRequired
AuthorizationBearer <EXTERNAL_API_KEY>Required
Content-Typeapplication/json or multipart/form-dataRequired
X-User-Emailuser@example.comOptional

Option 1 — JSON Body application/json

NameTypeRequiredDescription
querystringRequiredNatural language question
branchstringOptionalADO branch to search. Use /api/branches to list available branches
screenshotobjectOptionalCora PPM UI screenshot. Claude extracts page context and appends it to the query. Omit entirely to submit without a screenshot
screenshot.image_b64stringRequired if screenshot providedBase64-encoded image data (PNG, JPEG, WEBP, or GIF)
screenshot.mime_typestringOptionalImage MIME type. Default: image/png

Option 2 — File Upload multipart/form-data

NameTypeRequiredDescription
querytext fieldRequiredNatural language question
branchtext fieldOptionalADO branch to search
screenshotfileOptionalImage file (PNG, JPEG, WEBP, or GIF) — uploaded directly, no base64 needed
curl example
curl -X POST /api/query \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "X-User-Email: jane@company.com" \
  -F "query=Why is the EAC Submit button greyed out?" \
  -F "branch=supported_release-1.25.2" \
  -F "screenshot=@screenshot.png"

Response

202 Accepted
{
  "job_id": "a1b2c3d4-...",
  "status": "processing",
  "poll_url": "/api/query/a1b2c3d4-...",
  "message": "Query submitted. Poll poll_url for results.",
  "screenshot_context": "Project Forecasting page — EAC Submit button visible..."
}

screenshot_context is only included when a screenshot was provided. It shows the extracted UI context that was appended to your query.

GET /api/query/{job_id} Poll async query result Bearer

Polls for the result of an async query started via /api/query. Returns the result when complete, or a processing status while running. Use optional query parameters to return only specific parts of the response.

Headers

NameValueRequired
AuthorizationBearer <EXTERNAL_API_KEY>Required

Path Parameters

NameTypeRequiredDescription
job_idstringRequiredJob ID returned by POST /api/query

Query Parameters

NameTypeRequiredDescription
fieldsstringOptionalComma-separated list of top-level fields to include. Valid values: answer, sections, metadata. Default: all fields.
sectionstringOptionalReturn a single section only. Valid values: overview, user_guide, technical_reference.

Response (completed — default)

200 OK
{
  "success": true,
  "job_id": "a1b2c3d4-...",
  "status": "completed",
  "answer": "## Overview\n...## User Guide\n...## Technical Reference\n...",
  "sections": {
    "overview": "...",
    "user_guide": "...",
    "technical_reference": "..."
  },
  "metadata": { ... }
}

Response (with ?section=overview)

200 OK
{
  "success": true,
  "job_id": "a1b2c3d4-...",
  "status": "completed",
  "section_name": "overview",
  "section": "Plain-English explanation of the feature..."
}

Response (with ?fields=sections,metadata)

200 OK
{
  "success": true,
  "job_id": "a1b2c3d4-...",
  "status": "completed",
  "sections": { ... },
  "metadata": { ... }
}

Response (still processing)

200 OK
{
  "job_id": "a1b2c3d4-...",
  "status": "processing",
  "message": "Search is still running. Poll again shortly."
}

GET /api/health API health check Public

Public health check endpoint for the external API. Returns whether the API key is configured and the service is ready.

Response

200 OK
{
  "status": "ok",
  "api_ready": true
}
GET /api/branches List available ADO branches Public

Returns a list of branches from the Azure DevOps repository that have been indexed for code search. Uses the Code Search API facets with a Git Refs API fallback.

Response

200 OK
{
  "success": true,
  "branches": [
    "supported_release-1.25.2",
    "supported_release-1.25.1",
    "develop"
  ]
}

API Key Management
GET /api/admin/api-keys List all API keys Admin

Returns all API keys with their metadata. Raw keys and hashes are never exposed — only the prefix (first 12 characters) is shown.

Response

200 OK
{
  "success": true,
  "keys": [
    {
      "id": "a1b2c3d4-...",
      "key_prefix": "cora_a1b2c3d4...",
      "label": "Jira Integration",
      "created_by": "admin@corasystems.com",
      "created_at": "2026-04-04T12:00:00Z",
      "last_used": "2026-04-04T14:30:00Z",
      "is_active": true
    }
  ]
}
POST /api/admin/api-keys Create a new API key Admin

Generates a new API key. The raw key is returned once in the response and can never be retrieved again — only a prefix is stored for display. Keys use cora_ prefix + 48 random hex characters.

Request Body

NameTypeRequiredDescription
labelstringRequiredHuman-readable label (e.g. "Jira Integration", "n8n Workflow")

Response

201 Created
{
  "success": true,
  "key_id": "a1b2c3d4-...",
  "api_key": "cora_8f3a1b2c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e",
  "key_prefix": "cora_8f3a1b2c...",
  "label": "Jira Integration",
  "message": "Save this key now — it cannot be retrieved again."
}
PATCH /api/admin/api-keys/{key_id} Revoke, reactivate, or rename a key Admin

Update an API key's status or label. Set is_active to false to revoke a key (it will immediately stop working). Set back to true to reactivate.

Request Body

NameTypeRequiredDescription
is_activebooleanOptionalSet false to revoke, true to reactivate
labelstringOptionalUpdate the key's label
