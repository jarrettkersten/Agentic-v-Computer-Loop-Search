Core Search

Returns a list of branches from the Azure DevOps repository that have been indexed for code search. Uses the Code Search API facets with a Git Refs API fallback.

Agentic Search
POST /api/agentic/stream SSE stream for agentic search Login

Starts an agentic search loop and streams progress events via Server-Sent Events (SSE). The AI iteratively searches the ADO codebase using tool calls (code search + file reading) until it has enough context to answer. Events include iteration progress, tool calls, pause prompts, and the final result.

Request Body

NameTypeRequiredDescription
querystringRequiredNatural language question about the Cora PPM codebase
branchstringOptionalADO branch to search. Defaults to latest supported_release branch

Response — SSE Events

text/event-stream
event: iteration
data: {"iteration": 1, "action": "Made 2 ADO tool call(s)."}

event: tool_call
data: {"tool": "ado_code_search", "input": {"search_query": "TimesheetGrid"}}

event: pause
data: {"elapsed_sec": 302, "message": "Search has been running for 5 min..."}

event: done
data: {"answer": "## Overview\n...", "confidence": {"level": "high"}, ...}

Try it out

Note: This endpoint returns an SSE stream. The try-it panel will show the raw streamed events.

POST /api/agentic/start Start async agentic job Login

Starts an agentic search as a background job and returns a job ID. Use /api/agentic/poll/<job_id> to retrieve progress events. This is the polling-based alternative to the SSE stream, useful when HTTP proxies enforce connection timeouts.

Request Body

NameTypeRequiredDescription
querystringRequiredNatural language question about the Cora PPM codebase
branchstringOptionalADO branch to search

Response

202 Accepted
{
  "job_id": "a1b2c3d4-e5f6-..."
}

Try it out

GET /api/agentic/poll/{job_id} Poll agentic job events Login

Retrieves queued SSE-style events from a running agentic job. Call repeatedly to drain the event queue. Returns an empty array when no new events are available. The final event will have event: "done".

Path Parameters

NameTypeRequiredDescription
job_idstringRequiredThe job ID returned by /api/agentic/start

Response

200 OK
{
  "events": [
    {"event": "iteration", "data": {"iteration": 1, "action": "Made 2 ADO tool call(s)."}},
    {"event": "done", "data": {"answer": "..."}}
  ]
}

Try it out

POST /api/agentic/continue Resume paused search Login

Signals a paused agentic search job to continue. The search pauses automatically at elapsed-time thresholds (default: 5 min first, then every 2 min) to let the user decide whether to keep searching.

Request Body

NameTypeRequiredDescription
job_idstringRequiredJob ID of the paused search

Response

200 OK
{"ok": true}

Try it out

POST /api/agentic/stop Stop running search Login

Signals a running agentic search job to stop immediately. The job will finalize with whatever context has been gathered so far.

Request Body

NameTypeRequiredDescription
job_idstringRequiredJob ID to stop

Response

200 OK
{"ok": true}

Try it out

POST /api/agentic Synchronous agentic search (legacy) Login

Legacy synchronous endpoint that runs the full agentic search loop and returns the complete result in a single JSON response. Can be long-running (up to 11 minutes). Prefer /api/agentic/start + polling for production use.

Request Body

NameTypeRequiredDescription
querystringRequiredNatural language question about the codebase
branchstringOptionalADO branch to search

Response

200 OK
{
  "success": true,
  "result": {
    "answer": "## Overview\n...",
    "confidence": {"level": "high", "reasoning": "..."},
    "total_iterations": 7,
    "total_searches": 12,
    "total_files_read": 15,
    "branch": "supported_release-1.25.2",
    "input_tokens": 45000,
    "output_tokens": 8500,
    "elapsed_seconds": 42.3
  }
}

Try it out

Warning: This endpoint may take several minutes to return. Consider using /api/agentic/start + polling instead.

Export & Feedback
POST /api/export Export search results as Word/PDF Login

Exports a search result as a formatted Word (.docx) or PDF document. The exported document includes the query, branch, metadata, and the full answer content organized into sections.

Request Body

NameTypeRequiredDescription
formatstringRequired"docx" or "pdf"
querystringRequiredOriginal search query
branchstringRequiredBranch that was searched
metadataobjectRequiredSearch metadata (iterations, files_read, elapsed_sec, etc.)
sectionsobject[]RequiredArray of {title, content} section objects

Response

Binary file download (application/vnd.openxmlformats-officedocument.wordprocessingml.document or application/pdf)

POST /api/flag Flag result as accurate/inaccurate Login

Allows users to flag search results as accurate or inaccurate, with optional explanation. Flags are stored in the database for admin review and quality tracking.

Request Body

NameTypeRequiredDescription
querystringRequiredOriginal query
loop_typestringRequiredType of search loop used
answerstringRequiredThe answer being flagged
flag_typestringRequired"accurate" or "inaccurate"
containerstringOptionalSource container/panel identifier
explanationstringOptionalUser explanation of why the result is flagged

Response

200 OK
{
  "success": true,
  "flag_id": 42
}

Try it out

POST /api/extract-screenshot Extract UI context from screenshot Public

Accepts a base64-encoded screenshot image and uses Claude to extract UI context (page name, visible fields, error messages, data shown). The extracted context is returned as a plain-text string that can be appended to user queries for more targeted searches.

Request Body

NameTypeRequiredDescription
image_b64stringRequiredBase64-encoded image data
mime_typestringOptionalImage MIME type. Default: image/png

Response

200 OK
{
  "success": true,
  "context": "This screenshot shows the Project Forecasting page with EAC columns visible..."
}

External API (v1)
POST /api/v1/query Unified query endpoint (sync/async) Bearer

The primary endpoint for external integrations (Jira, n8n, custom tools). Accepts a natural language query and returns the agentic search result. Supports both synchronous and asynchronous modes. Requires a Bearer token matching the EXTERNAL_API_KEY environment variable.

Headers

NameValueRequired
AuthorizationBearer <EXTERNAL_API_KEY>Required
Content-Typeapplication/jsonRequired

Request Body

NameTypeRequiredDescription
querystringRequiredNatural language question
branchstringOptionalADO branch to search
asyncbooleanOptionalIf true, returns a job_id immediately (202). Default: false

Response (sync)

200 OK
{
  "success": true,
  "answer": "## Overview\n...",
  "metadata": {
    "branch": "supported_release-1.25.2",
    "files_read": 18,
    "iterations": 8,
    "searches": 14,
    "confidence": "high",
    "elapsed_seconds": 47.2,
    "input_tokens": 52000,
    "output_tokens": 9200
  }
}

Response (async)

202 Accepted
{
  "job_id": "a1b2c3d4-...",
  "poll_url": "/api/v1/query/a1b2c3d4-..."
}

Try it out

Authentication
GET /login Initiate Entra ID OAuth flow Public

Redirects the user to Microsoft Entra ID (Azure AD) login page. Disabled when AUTH_DISABLED=true or no ENTRA_CLIENT_ID is set. On success, redirects back to /auth/callback.

Response: 302 redirect to Microsoft login

GET /auth/callback OAuth callback handler Public

Handles the OAuth 2.0 authorization code callback from Microsoft Entra ID. Exchanges the code for tokens, fetches user profile via Microsoft Graph API, and stores user info in the Flask session.

Query Parameters (set by Microsoft)

NameTypeDescription
codestringAuthorization code from Entra ID
statestringCSRF state token

Response: 302 redirect to /

GET /logout Clear session & sign out Public

Clears the Flask session and redirects to Microsoft Entra ID logout. Ends the user's authenticated session completely.

Response: 302 redirect to Entra ID logout URL
